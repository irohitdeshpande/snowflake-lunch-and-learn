## Context

- **Database:** `MARKETING.PUBLIC.CAMPAIGN` table exists with columns: ID, TYPE, COUNTRY, EFFECTIVENESS, CITY
- **Warehouse:** `COMPUTE_WH` available (X-Small, running)
- **Compute Pool:** `SYSTEM_COMPUTE_POOL_CPU` available for container runtime
- **No existing Streamlit apps** in the account
- Deployment via SQL: upload app file to a stage, create STREAMLIT object with container runtime, then push live version

## Implementation Steps

1. **Create a stage** in `MARKETING.PUBLIC` to host the Streamlit app files

2. **Write the Streamlit app** (`streamlit_app.py`) with:
   - Connect to Snowflake using `st.connection("snowflake")`
   - Query `MARKETING.PUBLIC.CAMPAIGN` for effectiveness by country (and type)
   - Render bar charts using `st.bar_chart` or Plotly for grouped bar graphs
   - Include filters/selectors for Type and City

3. **Upload the app file** to the stage using `PUT` (via SQL or equivalent)

4. **Create the STREAMLIT object** using:
   ```sql
   CREATE STREAMLIT Marketing.PUBLIC.campaign_dashboard
     FROM '@Marketing.PUBLIC.streamlit_stage/app'
     MAIN_FILE = 'streamlit_app.py'
     RUNTIME_NAME = 'SYSTEM$ST_CONTAINER_RUNTIME_PY3_11'
     COMPUTE_POOL = SYSTEM_COMPUTE_POOL_CPU
     QUERY_WAREHOUSE = COMPUTE_WH;
   ```

5. **Push live version:**
   ```sql
   ALTER STREAMLIT Marketing.PUBLIC.campaign_dashboard ADD LIVE VERSION FROM LAST;
   ```

6. **Verify** by describing the Streamlit app and confirming it's deployed
